# learn_pcl

`learn_pcl` is a C++ project for learning and practicing `PCL (Point Cloud Library)`.
The repository currently focuses on a full point-cloud workflow: basic point-cloud operations, PCD IO, KITTI point-cloud loading, visualization, BEV export, and Web-friendly JSON export.

The project has also evolved from several standalone examples into a cleaner layered structure based on `src`, `src/apps`, `src/common`, and `include`.

## Goals

This repository currently covers the following topics:

- Basic `PCL` point-cloud data structures and algorithms
- Reading and writing `.pcd` files
- Loading KITTI `.bin` point clouds
- Voxel downsampling and basic preprocessing
- Point-cloud visualization and screenshot export
- Converting point clouds into `BEV` images
- Converting point clouds into Web-friendly `JSON`

## Development Environment

- OS: Linux
- Compiler: `g++` or `clang++` with `C++17` support
- Build tool: `CMake >= 3.20`
- Python: `python3` for the lightweight Web server

## Dependencies

Dependencies are managed by `vcpkg.json`.
The main libraries used by this project are:

- `pcl`
- `eigen3`
- `boost`
- `nlohmann-json`
- `opencv4`

The repository contains its own `vcpkg/` checkout and `vcpkg_installed/` directory so builds can be more reproducible.

## Repository Structure

The repository now follows a layered layout.

### Entry layer: `src/*.cpp`

These files are thin executable entry points. They mainly create and run the corresponding App classes.

- `src/main.cpp`
- `src/lesson_01_io.cpp`
- `src/lesson_kitti_io.cpp`
- `src/lesson_visualization.cpp`
- `src/pcd2bev.cpp`
- `src/pcd2web.cpp`

### Application layer: `src/apps/` and `include/apps/`

This layer contains the main logic for each demo or tool.

- `simple_cloud_app`
  Basic PCL example that creates a small `PointXYZ` cloud and computes its centroid.

- `lesson_01_io_app`
  Minimal `.pcd` write/read example.

- `lesson_kitti_io_app`
  Loads KITTI `.bin` point clouds and applies voxel downsampling.

- `lesson_visualization_app`
  Loads point clouds, visualizes them, and exports screenshots.

- `pcd2bev_app`
  Converts point clouds into BEV images and related feature maps.

- `pcd2web_app`
  Converts `.bin` or `.pcd` point clouds into JSON for browser-based viewing.

### Shared module layer: `src/common/` and `include/common/`

This layer contains reusable utilities shared by multiple executables.

- `point_cloud_io`
  Unified point-cloud loading helper.
  Currently supports:
  - KITTI `.bin`
  - `.pcd`

This module is reused by `lesson_kitti_io`, `lesson_visualization`, `pcd2bev`, and `pcd2web`.

## Directory Guide

### Root files

- `CMakeLists.txt`
  Main build file. Declares dependencies, include paths, and executable targets.

- `README.md`
  Project documentation.

- `vcpkg.json`
  Dependency manifest.

### `include/`

Header files organized by module.

- `include/apps/`
  Public declarations for App classes.

- `include/common/`
  Public declarations for shared utilities.

### `src/`

Source files organized into three parts:

- `src/*.cpp`
  Executable entry points.

- `src/apps/`
  Main implementations for demos and tools.

- `src/common/`
  Shared reusable implementations.

### `config/`

Runtime configuration files.

- `config/pointcloud.json`
  Central configuration file for point-cloud tools.
  It includes settings such as:
  - input point-cloud path
  - voxel leaf size
  - screenshot output path
  - BEV image output path
  - Web JSON output path
  - Web server port
  - BEV ranges and resolution

### `data/`

Example data files.

- `data/lesson_01_io.pcd`
  Example PCD file generated by `lesson_01_io`.

### `output/`

Generated outputs. Typical files currently include:

- `kitti_000000_bev.png`
- `kitti_000000_bev_height.png`
- `kitti_000000_bev_intensity.png`
- `kitti_000000_bev_density.png`
- `web_runtime_pointcloud.json`

### `scripts/`

Helper scripts.

- `scripts/build.sh`
  One-command build script.

- `scripts/serve_web.py`
  Lightweight Python Web server for static files and point-cloud conversion requests.

### `web/`

Browser-based point-cloud viewer.

- `web/index.html`
  Frontend entry page for viewing converted point clouds in a browser.

### `build/`

Generated build output directory. It contains:

- CMake-generated files
- compiled executables
- intermediate build artifacts

This is a generated directory, not a source directory.

### `vcpkg/`

In-repo `vcpkg` checkout used as the toolchain source.

Benefits:

- more reproducible dependency management
- easier environment setup
- less reliance on globally installed packages

### `vcpkg_installed/`

Installed dependency tree generated by `vcpkg`.

### `.local/`

Local helper tools used by the repository.
It currently contains tools such as `autoconf` used to smooth over dependency builds in some environments.

### `notes/`

Reserved directory for study notes, experiments, and troubleshooting records.

## Program Overview

### 1. `pcl_basic`

Introductory PCL example:

- manually creates a point cloud
- computes the 3D centroid
- prints cloud size and centroid

Useful for understanding:

- `pcl::PointCloud<T>`
- `PointXYZ`
- `pcl::compute3DCentroid`

### 2. `lesson_01_io`

Minimal `.pcd` IO example:

- generates a small point cloud
- writes it to `data/lesson_01_io.pcd`
- reads it back and prints sample data

### 3. `lesson_kitti_io`

KITTI point-cloud loading example:

- reads the input path from config
- loads `.bin` data into `PointXYZI`
- applies voxel downsampling
- reports point counts before and after filtering

### 4. `lesson_visualization`

Point-cloud visualization example:

- loads and downsamples a point cloud
- visualizes it with `PCLVisualizer`
- exports a screenshot according to config

### 5. `pcd2bev`

Point-cloud to BEV conversion tool:

- loads a point cloud
- crops it to configured ranges
- builds height, intensity, and density maps
- merges them into a BEV image
- writes results into `output/`

### 6. `pcd2web`

Point-cloud to Web JSON conversion tool:

- supports both `.bin` and `.pcd`
- supports command-line overrides
- supports voxel filtering and spatial range filtering
- outputs JSON that can be loaded by the frontend viewer

## Suggested Learning Order

A simple progression is:

1. `pcl_basic`
   Learn the most basic point-cloud container and centroid computation.

2. `lesson_01_io`
   Learn basic PCD read/write operations.

3. `lesson_kitti_io`
   Learn how real LiDAR point clouds are loaded and filtered.

4. `lesson_visualization`
   Learn how to inspect point clouds visually.

5. `pcd2bev`
   Learn how point clouds are converted into grid-based BEV representations.

6. `pcd2web`
   Learn how point clouds can be exported into browser-friendly data.

## Current Architecture Notes

Compared with the earlier single-file examples, the repository now has two clear improvements:

- each executable has its main logic isolated in an App class
- shared point-cloud loading logic has been extracted into a reusable common module

That makes the project easier to extend, easier to maintain, and more suitable as a structured PCL learning workspace.
